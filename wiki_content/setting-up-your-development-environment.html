Setting Up Your Development Environment

Developing Web pages and sites is not difficult, but having the right development environment is essential to success in building sites. 

At the end of the lesson, you should be able to:

  • understand and work with a three-tired web development environment: development machine, testing server, and production server.
  • install and work with the essential tools / software used by web developers / designers: html editors, css editor, simple graphics editor or creators, and ftp software

 

To demonstrate your competency with these topics you will:

  • review and install an html editor of your choice
  • review and install install a css editor
  • read Lesson 1: Markup Language and Site Development Essentials  
  • Lesson 1: Online Exercises

    Setting up Git and GitHub

    Git is a program that tracks and shares changes to files. GitHub is a cloud resources for Git that lets you host code repositories and files, sharing your changes and others. Git and GitHub are separate technologies that work together.

    Git

    You can use a command line tool or a GUI client. I prefer command line tools because they are useful in all of your development work, but the tool you choose is up to you.

    Git for Windows project (CLI and GUI options): https://gitforwindows.org/ (Links to an external site.)

    Git options for Mac: 

    • Use homebrew (https://brew.sh/ (Links to an external site.)) the Mac package manager to install git from the command line
    • XCode and the command line tools that go with it include git https://git-scm.com/download/mac (Links to an external site.)
    • Git Kraken is a great GUI for mac https://www.gitkraken.com/ (Links to an external site.)

    Git options for Linux

    • Ubuntu/Debian based linux: apt install git
    • Centos/Red Hat: yum install git
    • Git Kraken also works for linux https://www.gitkraken.com/

    GitHub

    1. Create an account with GitHub if you haven't already
    2. On your development computer create an ssh key. You must use ssh to communicate with GitHub. Anything else is insecure.
      1. Mac and Linux https://www.ssh.com/academy/ssh/keygen (Links to an external site.)
      2. Windows with PuTTYgen https://www.ssh.com/academy/ssh/putty/windows/puttygen (Links to an external site.) 
    3. Add your public key to your GitHub account https://docs.github.com/en/authentication/connecting-to-github-with-ssh/adding-a-new-ssh-key-to-your-github-account (Links to an external site.) 
    4. Use git!
