Server-Side Languages

Lesson Objective:  At the end of this lesson, you will be able to:

  • Compare popular client-side and server-side programming languages, including JavaScript, Java, PHP, Python, .Net, C, C++, Visual Basic, C#.
  • Define Common Gateway Interface (CGI) methods, including .Net, Django, Python, JavaServer Pages (JSP), Server-Side JavaScript (SSJS), Active Server Pages (ASP), PHP Hypertext Preprocessor (PHP), Ajax.
  • Identify ways to use additional technologies to provide custom features to an end user (e.g., using JavaScript to detect Web browser type, using cookies).


There are many server-side languages in use today, and many legacy server-side languages still used on the Web.  Key to understanding how server-side languages work is understanding that when you request a page that is written in one of these languages, the server sends the page through a rendering engine, either a .dll or .exe file, and the page is returned to you in client-side languages. 

Some of the more popular server-side scripts and languages include:

  • PHP
  • Classic ASP (using VBScript) and ASP.NET (Using any of a number of compiled programming languages) including:
    • C / C++
    • C#
    • VB.NET
    • Java
  • Java, JSP (Java Server pages), and Servlets
  • Perl (Practical Extraction and Report Language)
  • SGML (Server-side includes)
A term not used as widely now, that was in the past is CGI (common gateway interface).  PHP, Perl, Classic ASP, are all examples of CGI programming.
