Course Overview

This class is an introduction to web application development. It is a hands-on, project-oriented class that introduces students to fundamentals of programming for the web, such as understanding the major components of websites, basics of programming languages for the web (like PHP, JavaScript), and hands-on training to build a database-driven web application. In addition, the class includes discussions and considerations of the ethics of web-application design and development and takes a critical look at contemporary web apps.

Though we are specifically focused on JavaScript web app frameworks, the concepts taught in this course have applications across many platforms.

Essential Learning Outcomes

  • Understand the basic concepts of web development and JavaScript programming.
  • Apply basic web development concepts to a variety of problems.
  • Understand workflows and best practices for working in a software development team
  • Demonstrate understanding of source control systems

Upon successful completion of this course, students will be able to:

  • Envision the essential technological components of a web application idea.
  • Display understanding of full stack web development.
  • Collaborate with other teammates on a team software project.
  • Students will be prepared to leverage web resources in a variety of different projects.

Code Quality

Writing code that works is only part of what makes a good software developer. If code can’t be read, debugged, or extended then it isn’t useful. I will be grading on code quality, but you don’t have to get it right the first time. After I give feedback, I will allow re-submission.

The best way to ensure that your code meets these standards is to follow these guidelines.

  1. Format your code consistently. Indentation, line breaks and consistent spacing are all critical to readability.
  2. Make sure that logic statements (e.g. if (a<b)) are as simple as possible.
  3. Use descriptive, self-explanatory, names for your variables, functions and classes. For instance, if you have a function that adds two variables, name it ‘addTwoNumbers’ instead of ‘a’. I WILL NOT ACCEPT ONE-LETTER NAMES!!!
  4. Follow a style guide. I prefer either the AirBnb style guide or W3C style guide
  5. Use a linter like eslint. There will be an extra credit assignment to implement linting to demonstrate this in week 3.

Asking for Help

When you come to me for help with an assignment I will expect that you have first done your due diligence to research the issue. This includes:

  • Searching for an error message or output in google and stack overflow
  • post your issue on the "Discussion Group" on Canvas.
  • Working with your teammates to discover an answer
  • Read or researched the general concept through a tutorial or one of the resources I've provided
  • Be sure that you can describe the behavior you've observed and the steps you've tried so far.
  • If you want help with some portion of your app, make sure the code you wish me to review is available in GitHub.

Learning how to ask questions and search for answers is critical.

At no point will I accept screenshots of code as a valid debugging tool. Any code submitted for review must be checked into Git.

Academic Honesty

Using code from other repositories and developers is common practice in professional software development, but there is a difference between using and adapting other’s code and copying code without understanding it. It is critical to your learning that you understand the code and frameworks you use.

Reuse of code produced by other developers without attribution is plagiarism and, in some cases, a crime. Copying and pasting code from Stack Overflow or elsewhere without adaptation or comprehension is plagiarism. Any act of plagiarism will result in significant penalties to the student’s final grade. 

As long as you acknowledge the source and you are able to explain the source code in your own words and the modifications you made to be able to solve the task, you are OK.
