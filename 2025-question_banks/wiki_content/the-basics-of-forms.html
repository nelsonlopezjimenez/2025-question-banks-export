Lecture notes on forms

Forms, fields and elements An HTML form is a section of a document containing normal content, markup, special elements called controls (checkboxes, radio buttons, menus, etc.), and labels on those controls. Users generally "complete" a form by modifying its controls (entering text selecting menu items, etc.), before submitting the form to an agent for processing (e.g., to a Web server, to a mail server, etc.)
  1. Forms became a part of HTML in the 2.0 Standards.
  2. The form element
    • The form element should be contained within the BODY element of your document
    • The form element is a container tag, it requires an open and close statement, on and off!
    • All remaining portions of the form are contained with the form element.
      Example:

<form action=" " method" " id=" " enctype=" " target=" " onsubmit=" " onreset=" ">

The control elements of the form

</form>
  1. Attributes to the <form> tag or element are:
    • The action attribute
      1. This is a required attribute.
      2. This attribute specifies a form processing agent. For example, the value might be a URL (to submit the form to a program) or a mailto: URL (to email the form).
        Example: (submission to a program)

<form action="/cgi-bin/test.exe">

The control elements of the form

</form>
    Example: (submission by the mailto protocol/email)

<form action="mailto:persons_name@address.com">

The control elements of the form

</form>
  • The method attribute.
    1. This is a optional attribute.
    2. This attribute specifies which HTTP method will be used to submit the forms information.
    3. Acceptable (case-insensitive) values are "get" (the default) and "post".
      • Because this is an optional attribute, if you don't include it in your form element, the browser will assume "get"
      • While "get" is the default, "post" is the preferred value assigned to this attribute.
      • "get" tacks all the information acquired by the form onto the URL or mailto: of the action attribute, therefore, use the "post" method which separates the data.
        Example:

<form action="mailto:persons_name@address.com" method="post">

The control elements of the form

</form>
  • The id attribute
    1. This attribute assigns a unique identification to an element.
    2. This name must be unique in a document
      Example:

<form action="mailto:persons_name@address.com" method="post" id="guestbook">

The control elements of the form

</form>
  • The enctype attribute
    1. enctype tells the browser how it should treat or encode information being submitted from the form to the server, when the value of method is "post". This is called "content type"
    2. ENCYTPE can be MIME types or other types.
  • Encryption/MIME is beyond the scope of this class, if you want to learn more about it, visit the W3C page about MIME. For this class we will be using the least common denominator:
    Example:

<form action="mailto:persons_name@address.com" method="post" id="guestbook" enctype="text/plain">

The control elements of the form

</form>
  • The target attribute
    1. This attribute specifies the name of a frame where the form submission is to be opened.
    2. Acceptable values are a frame name, or one of the reserved values:
      "_self", "_parent", "_top" or "_blank".
      Example:

<form action="mailto:persons_name@address.com" method="post" id="guestbook" enctype="text/plain" target="_self">

The control elements of the form

</form>
  • The onsubmit and onreset attributes
    1. When a form is sent to a server for action, either to a program or other Internet protocol, the form is being "submitted", the onsubmit attribute is activated
    2. If the user has the option to "reset" the form with a button, the onreset attribute is activated.
    3. These attributes are used with scripting, either JavaScript, VBScript, JScript or Perl scripting.
  • The form element does nothing by itself, it needs control elements to allow interaction with the user.
++++++++++++++++++++++
  1. The input element
    • The input element is an empty tag, requiring only an on statement
    • It should be contained within the form element of your document.
      Example:

<input type=" " name=" " [additional attributes depending the the value assigned to type] />
  • The input element does nothing without attributes.
  • I think of the input tag as just a warning to the browser, the real work is done by the type=" " attribute
  • The type attribute
    1. is a required attribute of the input tag
    2. it defines what the user of your page will be able to do with the form field.
    3. acceptable values to be assigned to the type attribute are:
      text | password | checkbox | radio | submit | reset | file | hidden | image | button
    4. After choosing the value you assign to type, there are a number of associated or other attributes, which work with each of the "typeS" you assign.
  • text, creates a single-line text input control. This is the default value to the attribute type=" ".
    Example:

<form action="example" method="post">
     <input type="text" name="exampleForm" value="an example of input type='text'" size="30" maxlength="20" />
</form>

Continuing with Forms, fields and elements

  1. The select element
    • is a container tag requiring on and off tags.
    • The select tag by itself the select tag does nothing (another one of those wonderful tags).
    • The select tag and it's associated tags and attributes, create either a drop down box, or a list of selectable items.
    Syntax:

<select [name=" " size=" " multiple="multiple" disabled="disabled" tabindex=" " onfocus=" " onblur=" " onchange=" "]>

</select>
  • Attributes of the select tag are:
    1. name=" "
      • any value you want to assign, used to identify the particular textAREA tag
    2. size=" "
      • this is an integer value representing the number of lines you want to display in your list.
      • Default value is 1.
    3. multiple
      • allows for multiple selection from the list, one or more.
      • select assumes one unless you specify more.
    4. disabled
      • a boolean value, either true or false
    5. tabindex=" "
      • this is an integer value representing the position in the tabbing order.
    6. onfocus=" "
      • scripting, when the select has focus
    7. onblur=" "
      • scripting, when the select has lost focus
    8. onchange=" "
      • scripting, when a change has been made in the select
  • But again, without additional tags, the select tag does NOTHING!
  • The option element
    • is a container tag, it requires on and optional off tags.
    Syntax:


     <option [value=" " selected="selected" disabled="disabled"]>Visible Information</option>
  • Attributes of the option tag are:
    1. selected
      • if selected used as an attribute, the item is pre-selected from the list.
    2. disabled
      • boolean value
    3. value=" "
      • a pre-assigned value to the option
Example:

<select size="4" name="operating_system">

     <option selected value="Win3X">Windows 3.X</option>
     <option value="Win95">Windows 95 or NT</option>
     <option value="MAC">MacIntosh</option>
     <option value="Unix">Unix</option>

</select>
